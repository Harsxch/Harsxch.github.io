🔧 Managing and Creating Multiple Spotify Playlists Using Terraform

This project demonstrates how to use Terraform to create and manage multiple Spotify playlists programmatically. Below is a brief summary of the key points and workflow:

  • Prerequisites:
    • Spotify Developer account to obtain Client ID and Client Secret.
    • Terraform installed locally.
    • Docker (if using an auth proxy to handle Spotify OAuth).
  • Project Setup:
    • Clone the repository from GitHub.
    • Create a .env file (or export environment variables) with your Spotify credentials.
    • Set or export variables (like SPOTIFY_CLIENT_ID, SPOTIFY_CLIENT_SECRET, and SPOTIFY_API_KEY if needed).
  • Authentication Proxy (Optional):
    • Run docker run -it -p 27228:27228 --env-file .env ghcr.io/conradludgate/spotify-auth-proxy to handle local OAuth.
    • Ensure it’s accessible at http://localhost:27228.
  • Terraform Workflow:
    • In your Terraform code, define multiple spotify_playlist resources for each playlist.
    • Initialize with terraform init.
    • Review changes with terraform plan.
    • Apply the configuration with terraform apply to create or update playlists.
  • Result:
    • Terraform will create/update all specified playlists in your Spotify account.
    • Tracks, descriptions, and other properties can be managed via Terraform code.
    • Useful for quickly spinning up multiple playlists or syncing changes automatically.

This approach streamlines playlist management by leveraging Infrastructure as Code principles with Terraform.
